## 5. תוכנית ביצוע — 5 ימים לגמר

### יום 1 — תשתית Production

1. מילוי כל משתני [VERCEL-ENV-CHECKLIST](./VERCEL-ENV-CHECKLIST.md) ל-Production ו-Preview.  
2. `npm run db:migrate:deploy` (או `db:migrate:prod`) — וידוא שתי מיגרציות ה-embeddings/KV.  
3. וידוא 13 crons ב-Vercel Dashboard מול RUNBOOK.  
4. Sentry: deploy + אירוע בדיקה. PostHog: מפתח prod.  
5. בדיקת smoke: `/api/auth/session`, התחברות, טעינת workspace.

**שער יציאה מיום 1:** DB מעודכן, אין 5xx מיידיים אחרי login, crons מופיעים ב-dashboard.

### יום 2 — QA מסעות ליבה + Phase 3–4

#### 5.1 חמשת מסעות הליבה (חובה)

| # | מסע | קריטריון Pass |
|---|-----|---------------|
| 1 | התחברות credentials / passkey → workspace | ללא שגיאה, חלונות נטענים |
| 2 | פיננסים מ-quick grid → overview / cashflow | טאבים עובדים |
| 3 | מסמכים → סריקה או ארכיון | פתיחה + פעולה בסיסית |
| 4 | Google Drive מסרגל צד | חיבור או הודעת OAuth תקינה (לא crash) |
| 5 | קופיילוט שטח — יצירת סשן | אחרי migrate — ללא שגיאה |

#### 5.2 בנוסף (מומלץ באותו יום)

- Growth: blog / contact / lead  
- BOQ agent + accounting export (אם לא רצים רק ב-CI)  
- Knowledge Vault חיפוש סמנטי  
- דף מוצר: צילומים אם חסרים — `npm run product-brochure:capture` ([צעדים ידניים](./PRODUCT-BROCHURE-צעדים-ידניים.md))

**שער יציאה מיום 2:** 5/5 ליבה + רישום תוצאות ב-KPI.

### יום 3 — ביצועים, נגישות, אמינות

```bash
npm run verify
npm run test:e2e:ci-gate
npm run lighthouse:matrix:prod
# או: npm run lighthouse:sample  מול URL הפרוד
```

- axe: לוודא 0 Critical/Serious על workspace shell.  
- Sentry: תיעוד אחוז 5xx.  
- אופציונלי: `node scripts/load-test.mjs` עם `BASE_URL=https://www.bsd-ybm.co.il`.

**שער יציאה מיום 3:** Lighthouse prod ≥90; מספרי אמינות בטבלת KPI.

### יום 4 — אבטחה, CSP, DR, משפטי

1. Preview: `CSP_STRICT=true` → מעבר על [csp-production-checklist](./csp-production-checklist.md).  
2. אם ירוק — Production.  
3. Neon Console: תרגיל PITR (שחזור branch) — עדכון DR-PLAN.  
4. OAuth / כתובת משפטית / Resend — לפי runbooks.  
5. אימות `https://www.bsd-ybm.co.il/.well-known/assetlinks.json`.

**שער יציאה מיום 4:** CSP strict בפרוד או תוכנית WAIVE מתועדת; PITR מסומן; legal/OAuth במסלול.

### יום 5 — Sign-off וגמר

1. עדכון [PRODUCT-MAP](./PRODUCT-MAP.md) + Onboarding checkbox.  
2. מילוי מלא של [KPI-SIGNOFF](./KPI-SIGNOFF.md) — מדידה, תאריך, **מאשר**.  
3. סימון [LAUNCH-CHECKLIST](./LAUNCH-CHECKLIST.md).  
4. הגדרת on-call + SLO alerts.  
5. תיעוד קצר ב-PROGRESS: «גמר 10/10 — תאריך + קישורי מדידות».

**שער יציאה מיום 5:** אין תאים ריקים קריטיים ב-KPI; LAUNCH checklist ללא פריטי P0 פתוחים.
